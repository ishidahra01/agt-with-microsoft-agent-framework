# Demo Walkthrough

Use this skill when the operator wants a guided explanation of the demo rather than a raw execution result.

## Walkthrough Goals

- Explain each act in plain Japanese.
- Clarify what governance capability is being demonstrated.
- Connect the act result to a practical operational lesson.

## Act Framing

When relevant, organize the explanation around these acts:

1. Normal ticket review and safe next-action planning.
2. Blocking a risky follow-up request.
3. Detecting repeated risky behavior and escalating reliability controls.
4. Evaluating whether delegation to another agent is trustworthy.
5. Comparing MCP configurations before adoption.

## Recommended Output Structure

### Act N: タイトル

- 目的: what the act is meant to prove.
- 期待される結果: what should happen if governance is working.
- 実際の結果: what happened.
- 学び: why this matters operationally.

## Style

- Keep the narrative structured and concrete.
- Prefer short sections over long paragraphs.
- Emphasize that governance is enforced by runtime controls, not only by prompt wording.
